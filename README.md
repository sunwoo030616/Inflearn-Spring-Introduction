# 📝 스프링 입문 - 학습 기록

> **인프런 김영한 강사님의 '스프링 입문' 강의를 듣고 학습한 내용을 정리한 저장소입니다.**

## 📅 학습 기간

* 2026.01.18 ~ 2026.01.21 (완강)

## 🛠 사용 기술 (Tech Stack)

* **Language**: Java 17
* **Framework**: Spring Boot 3.x
* **Build Tool**: Gradle
* **DB**: H2 Database
* **Template Engine**: Thymeleaf
* **Library**: Spring Data JPA

## 📖 핵심 학습 내용

### 1. 스프링 웹 개발 기초

* **정적 컨텐츠**: 파일을 그대로 웹 브라우저에 전달
* **MVC와 템플릿 엔진**: 서버에서 변형을 가해 동적 HTML 전달 (Model, View, Controller)
* **API**: `@ResponseBody`를 사용하여 JSON 형식으로 데이터 전달

### 2. 회원 관리 예제 - 백엔드 개발

* **비즈니스 요구사항 정리**: 데이터(회원 ID, 이름), 기능(등록, 조회)
* **계층 구조**:
* **컨트롤러**: 웹 MVC의 컨트롤러 역할
* **서비스**: 핵심 비즈니스 로직 구현 (예: 중복 회원 가입 방지)
* **리포지토리**: 데이터베이스 접근, 도메인 객체 저장
* **도메인**: 회원 객체 등 핵심 비즈니스 객체



### 3. 스프링 빈과 의존관계 (DI)

* **컴포넌트 스캔**: `@Component` 애노테이션이 있으면 스프링 빈으로 자동 등록
* **자바 코드로 직접 등록**: `@Configuration`과 `@Bean`을 사용해 설정 파일에서 빈 등록
* **DI (Dependency Injection)**: 생성자 주입을 통해 객체 간의 의존관계를 스프링이 연결

### 4. DB 접근 기술의 발전

* **순수 Jdbc**: 초기 방식 (SQL 직접 작성, 연결 관리 복잡)
* **JdbcTemplate**: 반복 코드 제거, 하지만 SQL은 직접 작성
* **JPA**: 객체 지향적인 DB 프로그래밍 가능, SQL 쿼리 자동 생성
* **스프링 데이터 JPA**: 인터페이스만으로 공통 CRUD 및 핵심 기능 해결

### 5. AOP (Aspect Oriented Programming)

* **공통 관심 사항 vs 핵심 관심 사항** 분리
* 예: 모든 메소드의 호출 시간을 측정하는 로직을 별도의 클래스로 관리하여 핵심 로직을 깔끔하게 유지
